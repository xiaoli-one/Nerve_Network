## 2. 任务背景

在半导体制造过程中，晶圆缺陷检测是良率控制的重要环节。晶圆图谱记录了每个 die 的测试状态，不同缺陷类型会在晶圆空间分布上表现出不同模式，例如中心异常、边缘异常、划痕、局部聚集等。

本项目要解决的问题是：

```text
输入：一张晶圆 waferMap 矩阵
输出：该晶圆所属的缺陷类别
```

类别一共有 9 类：

| 编号 | 类别 | 含义 |
|---:|---|---|
| 0 | Center | 中心区域缺陷 |
| 1 | Donut | 环形缺陷 |
| 2 | Edge-Loc | 边缘局部缺陷 |
| 3 | Edge-Ring | 边缘环形缺陷 |
| 4 | Loc | 局部缺陷 |
| 5 | Near-full | 接近全片异常 |
| 6 | Random | 随机缺陷 |
| 7 | Scratch | 划痕缺陷 |
| 8 | none | 无明显缺陷 |

其中 `none` 类样本数量远大于其他缺陷类，因此这是一个明显类别不均衡的多分类问题。

## 3. 数据集说明

数据文件位于：

```bash
data/WM811K.pkl
```

原始数据为一个 pandas DataFrame，主要字段包括：

| 字段 | 说明 |
|---|---|
| `waferMap` | 晶圆图谱矩阵，每个元素表示 die 状态 |
| `failureType` | 缺陷类别标签 |
| `trainTestLabel` | 官方划分，取值通常为 `Training`、`Test` 或空标签 |
| `dieSize` | die 数量相关信息 |
| `lotName` | 批次名称 |
| `waferIndex` | 晶圆编号 |

### 3.1 有标注数据与无标注数据

WM811K 中并不是所有样本都有缺陷类别标签。本项目按 `trainTestLabel` 区分：

```python
mask = np.isin(split, ["Training", "Test"])
```

其中：

- `Training`：官方训练集，有标签；
- `Test`：官方测试集，有标签；
- 其他空标签样本：无标注数据。

程序默认只使用有标注数据。开启 `--use-unlabeled` 后，才会额外读取无标注样本并用于伪标签训练。

### 3.2 train / val / test 的来源

原始数据只有官方 `Training` 和 `Test`，没有官方验证集。因此代码从官方 `Training` 中再切分出验证集：

```text
官方 Training 的 85% -> train
官方 Training 的 15% -> val
官方 Test 的 100%    -> test
```

默认验证集比例由参数控制：

```bash
--val-ratio 0.15
```

三者作用如下：

| 数据集 | 来源 | 用途 |
|---|---|---|
| train | 官方 Training 的一部分 | 更新模型参数 |
| val | 官方 Training 中切出来的一部分 | 选择最佳 epoch、早停、调参 |
| test | 官方 Test | 最终性能评估 |

需要注意：`val` 和 `train` 都来自官方 Training，因此分布相似；`test` 是官方 Test，类别分布可能与训练集明显不同。

## 4. 数据预处理

原始 `waferMap` 是二维矩阵，元素通常为：

| 数值 | 含义 |
|---:|---|
| 0 | 晶圆外区域或无 die |
| 1 | 正常 die |
| 2 | 异常 die |

代码中核心预处理函数为：

```python
wafer_to_tensor(arr, size=96, augment=False)
```

### 4.1 补成正方形

不同晶圆的矩阵尺寸不完全一致。CNN 需要固定尺寸输入，因此先将矩阵居中补零为正方形：

```python
side = max(h, w)
square = np.zeros((side, side), dtype=np.uint8)
square[top : top + h, left : left + w] = arr
```

这样可以保持晶圆图谱的空间结构，同时避免拉伸变形。

### 4.2 one-hot 输入

代码没有直接把晶圆矩阵当作单通道灰度图，而是将 0、1、2 三种状态拆成 3 个通道：

```python
x = np.stack([square == 0, square == 1, square == 2]).astype(np.float32)
```

最终输入形状为：

```text
3 x image_size x image_size
```

使用 one-hot 的原因是：

- 0、1、2 是离散状态，不是连续灰度强度；
- one-hot 能让模型更清楚地区分晶圆外区域、正常 die、异常 die；
- 对晶圆缺陷图谱这类离散空间模式更合适。

### 4.3 resize

补成正方形后，使用最近邻插值调整到统一尺寸：

```python
F.interpolate(x[None], (size, size), mode="nearest")
```

默认尺寸：

```bash
--image-size 96
```

如果希望训练更快，可以降低到 64：

```bash
--image-size 64
```

如果希望保留更多细节，可以提高到 128：

```bash
--image-size 128
```

### 4.4 数据增强

训练集启用随机增强：

```python
arr = np.rot90(arr, random.randint(0, 3))
np.flip(arr, 0)
np.flip(arr, 1)
```

使用的增强包括：

- 随机旋转 0、90、180、270 度；
- 随机上下翻转；
- 随机左右翻转。

这些增强适合晶圆图谱，因为许多缺陷模式在旋转或翻转后仍然属于同一类别。增强的作用是：

- 增加训练样本多样性；
- 减少模型对固定方向的过拟合；
- 提高模型泛化能力。

验证集、测试集和伪标签预测阶段不使用随机增强。

## 5. 模型结构

代码提供两种模型：

```bash
--model fast
--model strong
```

默认使用：

```bash
--model fast
```

### 5.1 基础模块 ConvBNAct

代码中常用卷积模块为：

```python
Conv2d -> BatchNorm2d -> SiLU
```

作用：

- `Conv2d` 提取局部空间特征；
- `BatchNorm2d` 稳定训练、加快收敛；
- `SiLU` 作为非线性激活函数，通常比 ReLU 更平滑。

### 5.2 残差模块 ResBlock

残差模块结构为：

```text
输入 x
  -> ConvBNAct
  -> Conv2d + BatchNorm
  -> SEBlock
  -> Dropout2d
  + skip connection
  -> SiLU
```

残差连接的作用：

- 缓解深层网络梯度传播问题；
- 让模型更容易学习增量特征；
- 提高训练稳定性。

### 5.3 SE 注意力模块

SEBlock 全称为 Squeeze-and-Excitation Block。代码结构为：

```python
AdaptiveAvgPool2d(1)
Conv2d(ch, hidden, 1)
SiLU
Conv2d(hidden, ch, 1)
Sigmoid
```

作用：

- 对每个通道计算全局重要性；
- 自动增强重要通道；
- 抑制不重要通道；
- 提升模型对缺陷形态的表达能力。

在晶圆缺陷分类中，不同通道可能代表不同空间模式，SE 可以帮助模型关注更有判别力的特征。

### 5.4 fast 模型

fast 模型用于快速训练，单个 epoch 速度较快：

```text
输入 3 通道
-> 32 通道卷积
-> 32 通道卷积
-> MaxPool
-> 64 通道卷积
-> 64 通道卷积
-> MaxPool
-> ResBlock 64 -> 128
-> ResBlock 128 -> 128
-> Conv 128 -> 192
-> Global Average Pooling
-> Dropout
-> Linear 192 -> 9
```

特点：

- 通道最高到 192；
- 残差块数量少；
- 训练速度快；
- 适合作业快速实验和调参；
- 对总体 accuracy 较友好。

运行：

```bash
/Users/agiuser/py3/bin/python wm811k_cnn_assignment.py --model fast
```

### 5.5 strong 模型

strong 模型更深更宽，适合追求更高上限：

```text
输入 3 通道
-> 32 通道卷积
-> 64 通道卷积
-> ResBlock 64
-> ResBlock 64 -> 128
-> ResBlock 128
-> ResBlock 128 -> 256
-> ResBlock 256
-> ResBlock 256 -> 384
-> ResBlock 384
-> Global Average Pooling
-> Dropout
-> Linear 384 -> 9
```

特点：

- 通道最高到 384；
- 7 个残差块；
- 表达能力更强；
- 更有利于复杂缺陷类别；
- 单个 epoch 明显更慢。

运行：

```bash
/Users/agiuser/py3/bin/python wm811k_cnn_assignment.py --model strong
```

## 6. 训练策略

### 6.1 训练循环

每个 epoch 中，程序遍历训练集的所有 batch：

```text
读取 batch
-> 前向传播
-> 计算 loss
-> 反向传播
-> 梯度裁剪
-> 参数更新
-> 学习率调度
```

进度条中类似：

```text
361/361
```

表示当前 epoch 中共有 361 个 batch，已经完成 361 个。

batch 数量由训练样本数和 batch size 决定：

```text
batch 数 = ceil(train_size / batch_size)
```

例如：

```text
train = 46201
batch_size = 128
ceil(46201 / 128) = 361
```

### 6.2 优化器 AdamW

代码使用：

```python
torch.optim.AdamW(model.parameters(), lr=args.lr, weight_decay=1e-4)
```

AdamW 的特点：

- 自适应学习率；
- 对不同参数自动调整更新幅度；
- 使用 decoupled weight decay，正则化效果更稳定。

默认学习率：

```bash
--lr 3e-4
```

### 6.3 OneCycleLR 学习率调度

代码使用 OneCycleLR：

```python
torch.optim.lr_scheduler.OneCycleLR(...)
```

其思想是：

- 前期逐步升高学习率；
- 中后期逐步降低学习率；
- 有助于更快收敛；
- 有时能获得更好的泛化性能。

关键参数：

```python
pct_start=0.12
div_factor=20
final_div_factor=200
```

### 6.4 梯度裁剪

训练中使用：

```python
nn.utils.clip_grad_norm_(model.parameters(), 5)
```

作用：

- 防止梯度爆炸；
- 提高训练稳定性；
- 避免出现 NaN loss。

如果出现非有限 loss 或非有限梯度，代码会跳过该 batch：

```python
if not torch.isfinite(loss):
    skipped += 1
    continue
```

### 6.5 早停

代码中有早停机制：

```bash
--patience 12
```

如果验证集指标连续若干 epoch 没有提升，则提前停止训练。

作用：

- 防止过拟合；
- 节省训练时间；
- 自动选择较优 epoch。

### 6.6 最佳模型保存

每当验证集分数提升时，保存 checkpoint：

```python
torch.save({"model": best_state, "classes": CLASSES, "args": vars(args)}, "wm811k_best.pt")
```

最终测试前会加载验证集最好的模型：

```python
model.load_state_dict(best_state)
```

## 7. 不均衡类别处理

WM811K 数据集中 `none` 类远多于其他缺陷类，测试集尤其明显。因此代码提供两种训练目标模式：

```bash
--mode acc
--mode balanced
```

### 7.1 acc 模式

默认模式：

```bash
--mode acc
```

该模式目标是提高 overall accuracy。代码默认设置：

```python
sampler_power, weight_power, focal_gamma = 0.0, 0.0, 0.0
```

含义：

- 不使用 WeightedSampler；
- 不使用类别权重；
- FocalLoss 退化为接近普通交叉熵；
- 减少将 `none` 误判为缺陷类的情况。

适用场景：

- 作业主要看分类精度 accuracy；
- 希望测试集整体准确率更高；
- 接受少数类 F1 可能下降。

运行：

```bash
/Users/agiuser/py3/bin/python wm811k_cnn_assignment.py --mode acc
```

### 7.2 balanced 模式

balanced 模式更重视少数类缺陷：

```bash
--mode balanced
```

该模式设置：

```python
sampler_power, weight_power, focal_gamma = 0.75, 0.25, 1.2
```

启用方法包括：

- WeightedSampler；
- 类别权重；
- FocalLoss。

适用场景：

- 希望提高 Macro F1；
- 希望少数缺陷类表现更好；
- 接受 overall accuracy 可能略低。

运行：

```bash
/Users/agiuser/py3/bin/python wm811k_cnn_assignment.py --mode balanced --none-bias 0
```

### 7.3 WeightedSampler

WeightedSampler 的作用是改变训练 batch 中各类别出现的频率，让少数类更容易被采样到：

```python
WeightedRandomSampler(...)
```

其权重由类别计数反比计算：

```python
weights = (1 / counts ** power)
```

`power` 越大，少数类被增强得越明显。

### 7.4 类别权重

类别权重用于 loss 计算：

```python
F.cross_entropy(logits, y, weight=self.weight)
```

少数类拥有更大的 loss 权重，模型在少数类上犯错会受到更大惩罚。

### 7.5 FocalLoss

FocalLoss 形式为：

```text
loss = (1 - pt)^gamma * cross_entropy
```

其中 `pt` 是模型对真实类别的预测概率。

作用：

- 降低简单样本的 loss；
- 增强困难样本的影响；
- 常用于类别不均衡问题。

注意：FocalLoss 的 loss 数值通常不能与普通交叉熵直接比较。开启 FocalLoss 后，loss 可能会更低，但不一定代表 accuracy 更高。

## 8. none_bias 推理校准

代码提供参数：

```bash
--none-bias 0.35
```

作用是在推理时给 `none` 类的 logit 加一个偏置：

```python
out[:, CLASSES.index("none")] += none_bias
```

设计原因：

- 官方测试集中 `none` 占比很高；
- 模型如果过于偏向缺陷类，会将很多正常晶圆误判为缺陷；
- 给 `none` 增加适当偏置，可以提高 overall accuracy。

调参建议：

| 目标 | 参数建议 |
|---|---|
| 提高 overall accuracy | `--none-bias 0.35` 或 `0.5` |
| 平衡 accuracy 和 F1 | `--none-bias 0.2` |
| 关注少数类缺陷 | `--none-bias 0` |

需要注意：`none_bias` 是推理阶段校准，不改变训练数据，也不改变模型参数。

## 9. 测试时增强 TTA

TTA 全称 Test-Time Augmentation。代码支持：

```bash
--tta 1
--tta 4
--tta 8
```

实现方式：

- `tta=1`：原图推理一次；
- `tta=4`：旋转 0、90、180、270 度后分别推理，再平均 logits；
- `tta=8`：旋转 4 次 + 左右翻转后旋转 4 次，共 8 次推理。

代码逻辑：

```python
out = torch.stack(logits).mean(0)
```

优点：

- 推理更稳定；
- 降低模型对方向的敏感性；
- 可能提高测试准确率。

缺点：

- 推理速度变慢；
- `tta=4` 约为普通推理 4 倍计算量；
- `tta=8` 约为普通推理 8 倍计算量。

推荐：

```bash
--tta 4
```

如果追求最高结果且时间允许：

```bash
--tta 8
```

## 10. 无标注数据利用：伪标签半监督训练

作业中提到可以尝试使用无标注数据提升训练效果。当前代码已经实现了伪标签方法，默认关闭，需要手动开启：

```bash
--use-unlabeled
```

### 10.1 伪标签基本思想

伪标签 pseudo-labeling 的流程：

```text
1. 先用有标注 Training 数据训练模型
2. 用当前最佳模型预测无标注晶圆图
3. 只保留高置信度预测结果
4. 将这些预测类别当作伪标签
5. 把伪标签样本加入训练集继续训练
```

### 10.2 相关参数

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `--use-unlabeled` | 关闭 | 是否启用无标注伪标签训练 |
| `--pseudo-start-epoch` | 3 | 第几个 epoch 后开始生成伪标签 |
| `--pseudo-threshold` | 0.995 | 最大预测概率阈值 |
| `--max-pseudo` | 30000 | 最多加入多少伪标签样本，0 表示不限制 |
| `--pseudo-pool` | 100000 | 最多扫描多少无标注样本，0 表示扫描全部 |

### 10.3 推荐命令

保守使用：

```bash
/Users/agiuser/py3/bin/python wm811k_cnn_assignment.py \
  --use-unlabeled \
  --pseudo-start-epoch 3 \
  --pseudo-threshold 0.995 \
  --max-pseudo 30000 \
  --pseudo-pool 100000
```

更积极使用：

```bash
/Users/agiuser/py3/bin/python wm811k_cnn_assignment.py \
  --use-unlabeled \
  --pseudo-start-epoch 3 \
  --pseudo-threshold 0.98 \
  --max-pseudo 50000 \
  --pseudo-pool 200000
```

### 10.4 风险与注意事项

伪标签不是一定提升。如果阈值太低，错误伪标签会进入训练集，可能导致模型性能下降。

建议：

- 先用不带无标注数据的模型作为 baseline；
- 再开启 `--use-unlabeled` 对比；
- 阈值建议从 `0.995` 或 `0.98` 开始；
- 如果测试集 accuracy 下降，应提高阈值或减少 `--max-pseudo`。

## 11. 评价指标

代码输出三个主要指标：

```text
Accuracy
Macro F1
Weighted F1
```

### 11.1 Accuracy

Accuracy 是整体分类准确率：

```text
Accuracy = 预测正确样本数 / 总样本数
```

优点：

- 直观；
- 作业要求中明确提到分类精度；
- 适合作为总体性能指标。

缺点：

- 在类别极不均衡时容易被多数类支配；
- WM811K 测试集中 `none` 类很多，因此全预测 `none` 也能得到较高 accuracy。

代码会打印：

```text
测试集全预测 none 的 accuracy baseline
```

用于判断模型是否真正超过简单基线。

### 11.2 Precision / Recall / F1

对某一类：

```text
Precision = 预测为该类的样本中，有多少是真的
Recall    = 真实为该类的样本中，有多少被找出来
F1        = Precision 和 Recall 的调和平均
```

F1 公式：

```text
F1 = 2 * Precision * Recall / (Precision + Recall)
```

### 11.3 Macro F1

Macro F1 是所有类别 F1 的简单平均：

```text
Macro F1 = 9 个类别 F1 的平均值
```

特点：

- 每个类别权重一样；
- 更能反映少数类缺陷识别能力；
- 不会被 `none` 这种大类主导。

如果 Accuracy 高但 Macro F1 低，说明模型可能主要识别好了 `none`，少数缺陷类表现较弱。

### 11.4 Weighted F1

Weighted F1 按类别样本数加权：

```text
Weighted F1 = 各类别 F1 按 support 加权平均
```

特点：

- 比 Macro F1 更接近整体表现；

## 16. 为什么有时 loss 变高但 accuracy 变高

代码中 `acc` 模式和 `balanced` 模式使用的训练目标不同。

balanced 模式启用 FocalLoss：

```text
loss = (1 - pt)^gamma * cross_entropy
```

当样本很容易分类时，FocalLoss 会把该样本 loss 压得很小，因此 loss 数值可能很低。

acc 模式中：

```python
focal_gamma = 0.0
```

此时 loss 更接近普通交叉熵，不会大幅压低简单样本 loss。因此可能出现：

```text
loss 数值变高
accuracy 反而提高
```

这不矛盾，因为两个 loss 的度量口径不同。判断模型效果时，应主要看目标指标，例如：

- 如果目标是分类精度，看 Accuracy；
- 如果目标是少数类识别，看 Macro F1；
- 如果目标是总体加权表现，看 Weighted F1。

## 18. 性能与速度分析

影响训练速度的主要因素：

| 因素 | 影响 |
|---|---|
| `--model strong` | 模型更深更宽，单 epoch 明显变慢 |
| `--image-size` 增大 | 输入分辨率更高，计算量增加 |
| `--batch-size` 较小 | batch 数增多，epoch 时间变长 |
| `--num-workers 0` | 数据预处理串行执行 |
| 开启 `--use-unlabeled` | 需要额外扫描无标注数据 |
| `--tta 8` | 测试推理成本显著增加 |

提速建议：

```bash
--model fast
--image-size 64
--batch-size 256
--num-workers 4
--tta 1
```

需要注意：更快的配置可能会牺牲部分精度。

## 19. 局限性与改进方向

当前实现已经覆盖 CNN 分类、数据增强、注意力、TTA、半监督伪标签等方法，但仍有改进空间：

1. 预处理缓存  
   当前每个 epoch 都会动态执行补方形、one-hot、resize。可以将固定预处理结果缓存为 tensor，提高训练速度。

2. 更系统的 none_bias 校准  
   当前 `none_bias` 是手动设置，可以在验证集或单独校准集上搜索最优值。

3. 更稳健的半监督学习  
   当前伪标签只生成一次。可以改为多轮伪标签、自训练、teacher-student 或 EMA teacher。

4. 更强模型  
   可以尝试 EfficientNet、ResNet、ConvNeXt 等结构，但需要注意作业代码复杂度和训练时间。

5. 类别分布重校准  
   官方 test 与 training 分布不同，可考虑基于类别先验进行后验校准。

6. 多次随机种子实验  
   单次划分结果可能受随机种子影响，严谨实验可跑多个 seed 取平均。
